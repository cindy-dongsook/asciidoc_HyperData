[#sect_admin_auditlog]
=== Audit Log 

[시스템 관리] > [Audit Log] 메뉴를 선택하면 HyperData 내에서 발생한 작업들의 기록을 조회할 수 있다.
 
image::resources/figure_auditlog_main.png[title="Audit Log 화면", width=750]




* 검색조건
+
'기간', '시작 시간', '종료 시간', '프로젝트', '유형', '사용자 ID' 항목에 조회할 값을 입력한 후 [검색] 버튼을 클릭하면 조건에 맞는 Audit Log 기록을 선별해서 조회할 수 있다. '기간' 검색은 1시간, 6시간, 24시간, 1주, 1개월 중 선택 가능하다. 설정된 검색조건은 [underline]##초기화##를 클릭해서 초기화한다.
* 로그 목록
+
다음은 각 항목에 대한 설명이다.
+
[width="100%",cols="3,7a",options="header"]
|================
|항목 								|설명
|이벤트 발생 시간			|Action의 수행 날짜와 시간을 기록한다.
|프로젝트					|Action이 수행된 프로젝트를 나타낸다. 로그인 이력, 사용자 및 사용자 그룹
관리 등 프로젝트에 종속되지 않는 Action은 공란으로 표기된다.
|사용자 ID					|Action을 실행한 계정 정보를 기록한다. 
|유형						|HyperData Web UI에서 수행한 Action의 종류를 나타낸다. +
다음의 Action 종류를 제공한다. +
– CONNECTION : 관리자와 일반 사용자의 HyperData 로그인 이력을 기록
한다. +
– USER : 계정에 대한 CUD(Create, Update, Delete) 이력을 기록한다. +
– JOB : Data Job과 Service Job 관련 기록을 남긴다. Job에 대한 CUD(Create,
Update, Delete)를 수행하거나, Job을 실행한 경우 JOB Component로
Audit Log가 남는다. +
– FILE : 데이터를 다운로드 받는 경우, Job 수행 결과로 파일 쓰기를 진행한
경우, SAM 파일을 데이터소스로 등록하는 경우 이력을 남긴다. 다운로드
의 경우 해당 row를 클릭하면 다운로드 사유가 조회된다([그림 2.18] 참고). +
– PROJECT : 프로젝트에 대한 CUD(Create, Update, Delete) 이력을 기록
한다. +
– DATA : 데이터소스 및 오브젝트에 대한 CUD 이력을 기록한다.
|상세 설명					|Action에 대한 상세 설명이다.
|================
+
'유형' 항목이 'FILE'인 경우 해당 row를 클릭하면 다운로드 사유를 조회할 수 있다.
