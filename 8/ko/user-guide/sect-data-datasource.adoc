[#sect_data_datasource]
=== Data Source 

[데이터] > [Data Source] 메뉴를 선택하면 현재 프로젝트에 등록된 데이터소스와 객체 현황을 조회할 수 있다. 데이터소스 화면에서 #Default DataSource와 #SQL 등의 디폴트 데이터소스 외에도 외부 소스를 등록해서 관리할 수 있다.

NOTE: 본 메뉴는 생성된 프로젝트에 진입한 상태여야 활성화되고 HyperData Web UI에 나타난다.

[+데이터소스] 버튼을 클릭하면 외부 데이터소스를 등록할 수 있다. 데이터소스 트리에서 노드를 선택한 후 [삭제] 버튼을 클릭하면 객체와 외부 데이터소스를 삭제할 수 있다.
 
[#figure_data_datasource]
image::resources/figure_data_datasource.png[title="데이터소스 화면",width=750]

데이터소스 화면 트리 위의 '검색' 입력항목에 검색할 데이터소스, 데이터 객체, 컬럼을 입력하고 image:resources/icon_search.png[width=22] 아이콘을 클릭하거나 <Enter> 키를 누르면 트리 내부를 검색할 수 있다. 트리의 노드를 선택하면 해당 객체의 상세 정보가 오른쪽 영역에 조회된다.

[#figure_data_datasource_detail] 
image::resources/figure_data_datasource_detail.png[title="데이터소스 상세 화면",width=750]


* 데이터소스 트리
+
등록된 #Default DataSource와 #SQL, 외부 소스의 목록을 조회한다. 각 노드의 하위에 생성된 테이블 정보가 트리로 구성되어 조회된다.
* 상세조회
+
데이터소스 트리의 노드를 선택하면 오른쪽 영역에 해당 소스와 객체(테이블)에 대한 상세 정보가 조회된다.
+
다음은 각 트리 노드별 상세화면에 대한 설명이다.

[width="100%",cols="3,8a",options="header"]
|===================
|항목			|설명

|작성자			|데이터소스를 생성한 사용자 이름이다. 사용자 ID가 아닌 이름이 표시된다.
|생성/수정 날짜	|데이터소스가 생성된 날짜와 시간과 마지막으로 수정된 날짜와 시간이다.
|소스 타입		|등록된 데이터소스의 유형이다.
#Default DataSource의 경우 HyperData가 기본적으로 사용하는 Tibero로 조
회되고, DB 링크를 통해 데이터 객체를 생성하는 #SQL 소스는 유형이 공란
으로 나온다. 사용자가 생성한 데이터소스는 Tibero, Oracle, sam(File) 등 선
택된 데이터소스 종류에 대응되서 출력된다. +
|소스 타입 상세 정보|데이터소스를 생성한 방식에 따라 상이하게 나타난다. +
– 연결 문자열 사용 : 데이터소스를 생성하는 경우 연결 문자열 사용 여부를
보여준다. +
• yes : 연결 문자열을 사용하여 등록된 데이터소스임을 의미하며, 사용자
ID와 연결 문자열이 이어져서 상세 정보에 나타난다. +
• no : 연결 문자열을 쓰지 않고 직접 DB 정보를 입력하여 생성된 데이터
소스임을 의미하며, 사용자 ID, IP, 포트, SID가 이어진다. 연결 문자열은
Tibero가 설치된 $TB_HOME의 client/config 디렉터리 내 'tbdsn.tbr'에서
지정할 수 있다. +
– 사용자 ID : 해당 DB의 사용자를 의미하며, 데이터소스를 등록할 때 지정
한 ID가 나타난다. +
– 연결 문자열 : 연결 문자열 사용이 'yes'인 경우 나타나며, 해당 데이터소스
를 등록하는 경우 사용한 연결 문자열을 보여준다. +
연결 문자열 사용이 'no'인 경우 다음의 상세 정보가 더 조회된다. +
• IP : 해당 DB의 IP 주소이다. +
• 포트 : 해당 DB의 포트 번호이다. +
• SID : 해당 DB의 SID이다.
|설명				|데이터소스를 생성할 때 작성한 설명이다.
|===================

==== 데이터소스 등록

HyperData에서 등록 가능한 데이터소스 유형으로는 외부 Client DB, csv 등 SAM 파일, 하둡 분산형 파일 시스템(HDFS: Hadoop Distributed File System), Excel의 4가지가 있다.   

<<figure_data_datasource, 데이터소스 화면>>에서 [+데이터소스] 버튼을 클릭하면 데이터소스를 등록할 수 있다. 데이터소스 생성이 완료되면 왼쪽 데이터 트리에 생성된 데이터소스가 조회된다. 본 절에서는 각 데이터소스를 등록하는 방법을 설명한다.

[#sect_datasource_create_xls]
===== Excel(((Excel)))
 
Create Data Source 화면의 '데이터소스 선택' 항목에서 [New File] > [Excel]을 선택하면 Excel 파일을 데이터소스로 등록할 수 있다. 데이터소스로 엑셀 파일을 등록하면, 엑셀 내 각 시트를 하나의 데이터 객체로 저장할 수 있다. 항목을 입력한 후 [등록] 버튼을 클릭한다.
 
image::resources/figure_excel_source_create.png[title="데이터소스 등록 (Excel)",width=450]

다음은 입력 항목에 대한 설명이다. 설명되지 않은 항목에 대한 설명은 <<sect_datasource_create_sam,"sam (File)">>을 참고한다.

[width="100%",cols="3,7",options="header"]
|=========
|항목		|설명
|이름		|
|파일 경로  |
|=========

추가할 엑셀 파일 중 헤더가 지정되지 않은 시트가 있는 경우 [헤더를 추가할 시트 번호] 버튼을 클릭해서 헤더를 추가할 수 있다.

Include Header 화면에서 설정하는 시트 번호는 등록할 엑셀 파일 내 시트 이름과 무관하고, 왼쪽부터 1번이다. [+시트] 버튼을 클릭해서 헤더를 추가할 시트 번호를 입력할 수 있다. 체크박스를 선택한 후 [삭제] 버튼을 클릭하면 등록된 시트 번호를 삭제할 수 있다. 설정을 완료한 후 [적용] 버튼을 클릭하면 '헤더를 추가할 시트 번호' 항목에 설정된 시트 번호를 확인할 수 있다(<<figure_excel_source_header_entered, 데이터소스 등록 (Excel) - 시트에 헤더 추가 (2)>> 참고).

image::resources/figure_excel_source_header_include.png[title="데이터소스 등록 (Excel) - 시트에 헤더 추가 (1)",width=260]

'헤더를 추가할 시트 번호' 항목에 설정한 정보를 확인한 후 [등록] 버튼을 클릭한다. 해당 시트로 생성된 객체는 <COL0>, <COL1>, ...의 형태로 컬럼명이 자동으로 생성된다. 컬럼명은 데이터 객체를 생성할 때 <<figure_datasource_dataobject_create, 데이터 객체 생성>>에서 사용자가 원하는 값으로 변경할 수 있다.
 
[#figure_excel_source_header_entered]
image::resources/figure_excel_source_header_entered.png[ title="데이터소스 등록 (Excel) - 시트에 헤더 추가 (2)",width=450]

NOTE: 이미 헤더가 있는 시트에 [헤더를 추가할 시트 번호]를 지정한 경우 시트의 첫 번째 행에 <COL0>, <COL1>, ...의 형태로 헤더가 저장된다. [헤더를 추가할 시트 번호]를 설정하지 않은 경우는 객체를 생성할 때 첫 번째 행의 데이터를 헤더로 저장한다.

[#sect_datasource_create_HDFS_Directory}
===== HDFS 

Create Data Source 화면의 '데이터소스 선택' 항목에서 [New File] > [HDFS]을 선택하면 HyperData를 통해 Apache의 HDFS(Hadoop Distributed File System)에 저장된 데이터를 디렉터리 단위로 소스를 등록하고 관리할 수 있다.

 
image::resources/figure_datasource_hdfs_directory.png[title="데이터소스 등록 (HDFS)",width=450]

* 기본 정보
+
다음은 화면의 입력 항목에 대한 설명이다. 설명되지 않은 항목에 대한 설명은 <<sect_datasource_create_sam, "sam (File)">>을 참고한다.
+
[width="100%",cols="3,7",options="header"]
|==================
|항목				|설명
|HDFS 경로			|
|검색할 파일 확장자	|
|==================
* Kerberos Authentication
+
하둡 시스템이 커버로스 인증 프로토콜을 사용 중이면, SAM(HDFS)와 동일한 방법으로 커버로스 인증 프로토콜을 통해 하둡 파일시스템에 접근할 수 있다. 이 경우 'Kerberos Authentication' 항목을 선택해 추가 정보를 입력한다. 이 추가 정보에 대한 자세한 사항은 <<sect_datasource_create_sam_hdfs, "sam (HDFS)">>을 참고한다.

'HDFS 경로' 항목을 입력한 후 [HDFS 파일 선택] 버튼을 클릭하면 데이터 소스파일을 선택할 수 있다. Directory File Selector 화면에서 목록의 체크박스를 선택하고 파일 정보를 미리보기를 통해서 확인한 후 [적용] 버튼을 클릭한다.
 
[#figure_datasource_hdfs_directory_file_selector]
image::resources/figure_datasource_hdfs_directory_file_selector.png[ title="데이터소스 등록 (HDFS) - 디렉터리에서 파일 선택",width=750]

다음은 입력 항목에 대한 설명이다. 설명되지 않은 항목에 대한 설명은 <<sect_datasource_create_sam, "sam (File)">>을 참고한다.

[width="100%",cols="3,7",options="header"]
|================
|항목				|설명
|이름				|
|파일 경로			|
|크기				|
|타입				|
|마지막 수정 날짜   |
|구분자				|
|개행 문자			|
|헤더 생성			|

파일의 row를 선택한 후 조회할 행의 갯수를 입력하고 [보기] 버튼을 클릭하면 지정된 숫자만큼 앞서 지정된 구분자에 따라 row를 미리볼 수 있다. 현재 개행문자 \n, \r\n만 미리보기를 지원하고, 일반적인 csv 형태가 아닌 형식(xls, xlsx, jpg, jpeg, mp3, mp4, json, xml)은 미리보기를 지원하지 않는다.

[#sect_dataobject_detail]
==== 데이터 객체 상세 조회

<<figure_data_datasource, 데이터소스 화면>>의 데이터소스 트리에서 데이터 객체를 선택하면 [Details], [스키마], [데이터미리보기], [Downloads] 탭에서 객체의 정보를 조회할 수 있다.
 
[#figure_dataobject_detail]
image::resources/figure_dataobject_detail.png[title="데이터 객체 상세 조회",width=750]

* [Details] 탭
+
등록된 데이터 객체의 상세 정보를 조회하고 수정하거나 공유, 분석을 할 수 있다. [공유] 버튼을 클릭해서 생성한 데이터 객체를 프로젝트 간 상호 공유할 수 있다. [분석] 버튼을 클릭하면 분석 시나리오 추천 기능을 이용할 수 있다.
* [스키마] 탭
+
선택한 데이터 객체의 스키마 정보를 조회할 수 있다. SAM (File) 형식으로 등록한 데이터 객체는 [변환] 클릭해서 칼럼의 타입을 변경할 수 있다. 타입 변환이 완료되면 기존에 data flow와 visualization 차트에 등록해두었던 데이터 객체들을 재등록을 해주어야 한다.
* [데이터 미리보기] 탭
+
'행 개수 입력'에 조회할 행을 입력한 후 [보기] 버튼을 클릭하면 선택한 데이터 객체에 저장된 데이터를 조회할 수 있다. 데이터 미리보기는 100개의 행까지 확인이 가능하다.
* [Downloads] 탭
+
[Downloads] 탭에서 선택된 데이터 객체를 파일로 간편하게 저장할 수 있다. 다운로드된 파일 확장자는 csv이며, 데이터 객체의 내용을 파일로 출력할 때에 사용할 수 있다. 사용자는 다운로드한 csv 파일을 다시 새로운 데이터 객체로 업로드하여 HyperData에서 활용하거나 머신러닝 라이브러리의 input 파일 등으로 사용할 수 있다.
+
'컬럼 선택 옵션'과 '로우 선택 옵션'에서 원하는 컬럼과 조건을 입력하고, '출력 파일명'에 원하는 이름을 지정한 후 [Downloads] 버튼을 클릭하면, 해당 이름으로 csv 파일을 다운받을 수 있다.
+
[Downloads] 버튼을 클릭하면 데이터를 파일로 다운로드 받으려는 사유를 입력하는 화면이 나타난다. 사유를 입력한 후 [다운로드] 버튼을 클릭한다. 사유는 필수 항목으로 반드시 입력해야 다운로드 진행이 가능하다.
+
image::resources/figure_enter_reason_do_detail.png[title="데이터 객체 다운로드 - 사유서 입력",width=280]
+
다운로드가 완료되면 결과가 Audit Log에 다음의 형태로 저장된다.
+
....
"Data Object {데이터 객체 이름} downloaded as file {다운로드 파일 이름} (Explanatory Statement)."
....
+
기록된 로그 정보는 <<figure_auditlog_main, Audit Log 화면>>에서 조회할 수 있다. 로그의 유형은 'FILE'로 저장된다. 로그 목록의 row를 클릭하면 다운로드할 때 입력한 사유서를 조회할 수 있다.

[#sect_datasource_do_anaysis]
===== 데이터 객체 분석

<<figure_dataobject_detail, 데이터 객체 상세조회 화면>>의 [Details] 탭에서 [분석] 버튼을 클릭하면 분석 시나리오 추천 기능을 이용할 수 있다. <<figure_scenario_recommend_dialog,추천 분석 시나리오 화면>>에서 데이터소스 내의 테이블 정보를 분석하여 유의미한 분석 시나리오를 추천해 준다. 추천되는 시나리오는 최대 상위 10개이다.

[분석하기] 버튼을 클릭하면 데이터 소스 내의 테이블 정보를 분석하여 유의미한 분석 시나리오를 추천해 준다. 결과가 조회되기까지 시간이 소요되며, 소요되는 시간은 데이터의 크기에 따라 길어질 수 있다. 데이터 객체의 상세 조회 화면에서 [분석하기] 버튼을 클릭하면 [분석중]으로 버튼 텍스트와 활성상태가 변경된다.

서버 내에서 데이터객체의 분석 상황에 맞춰 텍스트와 활성상태가 변경되며 분석이 완료되면 [분석결과] 버튼으로 텍스트와 활성상태가 변경된다.

image::resources/figure_scenario_recommend_start.png[title="데이터 객체 상세 화면 - 분석 요청 및 진행 중",width=750]

[분석결과] 버튼을 클릭하면 추천 분석 시나리오 화면에서 분석 결과를 확인할 수 있다.

[#figure_scenario_recommend_dialog]
image::resources/figure_scenario_recommend_dialog.png[title="데이터 객체 상세조회 화면 - 분석",width=700]

추천 분석 시나리오 화면에서 왼쪽 시나리오 리스트에서 체크박스를 체크하면 [대시보드로 내보내기] 버튼이 활성화된다. 버튼을 클릭하면 대시보드로 내보내기 화면에서 시나리오 리스트에서 선택한 자료를 확인할 수 있으며 자료를 내보낼 대시보드를 선택하여 내보내기를 수행할 수 있다. 대시보드로 내보낸 시각화 자료는 <<figure_dashboard, 대시보드 화면>>의 목록에서 해당 대시보드를 클릭하면 조회할 수 있다.

[추천 분석 시나리오 보기] 버튼을 클릭하는 경우 이전 추천 분석 시나리오 화면으로 전환된다.

image::resources/figure_scenario_recommend_export.png[title="데이터 객체 상세조회 화면 - 분석 시나리오 내보내기",width=700]

[#sect_dataobject_column_edit]
==== 컬럼 정보 상세 조회 

<<figure_data_datasource,데이터소스 화면>>의 데이터소스 트리에서 데이터 객체의 하위 컬럼을 선택하면 상세화면에서 해당 컬럼의 상세 정보를 조회하고 수정할 수 있다.

image::resources/figure_dataobject_column_detail.png[title="컬럼 정보 수정",width=750]

[수정] 버튼을 클릭하면 컬럼을 수정할 수 있다. 컬럼 정보 수정 화면에서 항목을 수정한 후 [적용] 버튼을 클릭한다. '태그' 항목에 컬럼의 태그 정보를 입력하고 [+] 버튼을 클릭한다.
 
image::resources/figure_dataobject_column_edit.png[title="컬럼 정보 수정",width=600]
